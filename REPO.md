### What each file does

- **vm-compose.ps1**
  The CLI orchestrator. Handles `up`, `down`, `exec`, `docker`, `storage`, `health`, and more. Requires PowerShell 7+ and Administrator privileges for most commands.

- **vm-dashboard.ps1**
  Pode v2 web dashboard at `http://localhost:8080`. Shows VM status, storage table, and provides start/stop/restart and storage mount/detach actions. Install as a service with `vm-dashboard-install.ps1`.

- **vm-lib.ps1**
  Shared helper functions dot-sourced by `vm-compose.ps1` and loaded into all Pode route runspaces via `Use-PodeScript`. Handles VHD checkpoint chain detection (`.avhdx` parent walking) so storage lookups work correctly when VMs have checkpoints.

- **vm-metrics.ps1**
  Prometheus-compatible metrics exporter at `http://localhost:9090/metrics`. Exposes per-VM state, CPU, memory, uptime, IP, and Docker status. Install as a service with `vm-metrics-install.ps1`.

- **vmstack-example.yaml**
  A committed example config. Copy to `vmstack.yaml` and edit for your environment. See [QUICKSTART.md](QUICKSTART.md) for setup steps.

- **vmstack.yaml**
  Your live config ‚Äî gitignored so credentials and local paths don't leak.

# .gitignore

The `.gitignore` is tuned to keep the repo clean and free of secrets or large generated files:

- **Hyper-V VM artifacts** (`.vhd`, `.vhdx`, `.avhdx`, `.vmcx`, etc.) ‚Äî binary disk images, often hundreds of GB. The `storage/` folder where VHDXes live is excluded entirely.
- **ISOs** ‚Äî large OS install images, not source code.
- **Autogenerated setup files** (`Setup/`, `autounattend.vfd`) ‚Äî built on-the-fly by `vm-compose.ps1 up`; no value committing generated output.
- **Logs** (`.log`, `.evtx`) ‚Äî runtime output, not source.
- **PowerShell module files** (`.psd1`, `.psm1`, etc.) ‚Äî dependency artifacts, not part of this project.
- **Editor and OS junk** (`.vscode/`, `.DS_Store`, `Thumbs.db`, etc.)
- **`vmstack.yaml`** ‚Äî your live config may contain ISO paths, admin passwords, or other local details. Commit `vmstack-example.yaml` instead and document secrets separately.
- **`*-svc-wrapper.ps1`** ‚Äî service wrapper scripts are generated by the install scripts and contain local absolute paths.
- **Personal local files** (`local/`, `*.pw`) ‚Äî the `local/` directory holds any per-environment notes, keys, or reference files that should never be committed (passwords, host-specific docs, etc.). `*.pw` files (e.g. `solr.pw`) store VM admin passwords for quality-of-life use by scripts.
